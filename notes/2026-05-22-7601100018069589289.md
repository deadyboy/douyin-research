## 关键内容拆解

这个配置包不是零散的脚本或技巧集合，而是一个结构清晰、工程化的系统。它把Claude Code的能力拆解为几个核心模块：

- **agents/**：包含多个专业子代理，比如`planner.md`用于功能规划、`tdd-guide.md`用于测试驱动开发、`security-reviewer.md`用于安全审查等。
- **skills/**：定义了工作流和领域知识，如编码规范、前后端设计模式、持续学习机制等。
- **commands/**：提供可直接调用的命令，如`/plan`、`/tdd`，让AI代理能按指令执行特定任务。
- **rules/**：内置编码风格、安全检查等规则，让AI输出更规范、更安全。
- **mcp-configs/**：配置与外部服务（如GitHub、Supabase）的连接，实现自动化操作。

此外，作者还配套了两份指南：“Shorthand Guide”适合快速上手，“Longform Guide”深入讲解进阶技巧，比如上下文压缩、验证循环、持续学习等。

安装方式也设计得非常友好——通过Claude Code的插件市场一键添加，命令清晰明了，强调“新手友好”。

## 可以怎么复用

你可以从以下几个步骤开始尝试：

1. **安装插件**：在Claude Code中运行以下命令：
   ```
   /plugin marketplace add affaan-m/everything-claude-code
   /plugin install everything-claude-code@everything-claude-code
   ```
   安装后，尝试运行`/plan`或`/tdd`，看看AI是否能输出结构化的方案或测试用例。

2. **阅读指南**：访问项目推文或GitHub仓库中的`guides/`目录，阅读“Shorthand Guide”快速上手，再通过“Longform Guide”深入理解进阶技巧。

3. **启用规则**：将`rules/security.md`、`rules/coding-style.md`等文件复制到你的本地Claude配置目录（如`~/.claude/rules/`），观察AI是否自动遵循规范。

4. **尝试子代理**：用`/plan`命令描述一个功能需求，看AI是否能输出包含模块划分、技术选型、测试策略的完整方案。

5. **连接外部服务**：在`mcp-configs/mcp-servers.json`中配置GitHub，启用`/git`命令，尝试让AI直接提交PR或查看分支。
